# 020 — PWA (Progressive Web App)

> Doc d'entrée pour `specify.specify`. Feature bonus — mobile-first premium signal.

## Contexte / Objectif

Rendre Axiom installable comme une application native sur mobile et desktop via PWA, renforçant la perception de "vrai produit" au-delà d'un simple site web — cohérent avec le responsive mobile de la Phase 7.

## User Stories

- En tant qu'utilisateur mobile, j'installe Axiom sur mon écran d'accueil et l'ouvre comme une app native (sans barre d'adresse).
- En tant qu'utilisateur desktop, j'installe Axiom comme app desktop depuis Chrome/Edge.
- En tant qu'utilisateur, l'app affiche un écran de chargement on-brand (splash screen) au lancement.

## Exigences fonctionnelles

- **FR-2001** — Créer un `manifest.json` (via `app/manifest.ts` Next.js) : nom "Axiom", short_name "Axiom", description, start_url, display "standalone", background_color et theme_color conformes à la palette Axiom.
- **FR-2002** — Générer les icônes PWA nécessaires (192x192, 512x512) aux couleurs et wordmark Axiom.
- **FR-2003** — Implémenter un Service Worker minimal via `next-pwa` ou `@ducanh2912/next-pwa` pour le caching des assets statiques.
- **FR-2004** — Splash screen on-brand au lancement de l'app installée (background dark Axiom + wordmark centré).
- **FR-2005** — Bannière d'installation affichée sur mobile après X visites (comportement natif du navigateur — ne pas forcer une bannière custom intrusive).
- **FR-2006** — Vérification du score PWA via Lighthouse (PWA checklist verte).

## Critères d'acceptation

- Sur mobile Chrome, l'option "Ajouter à l'écran d'accueil" est disponible et installe l'app correctement.
- L'app installée s'ouvre en mode "standalone" (sans barre d'adresse navigateur).
- Le splash screen affiche le fond dark Axiom, pas un fond blanc par défaut.
- Le score Lighthouse PWA est vert (toutes les checks passent).

## Dépendances techniques

`next-pwa` ou `@ducanh2912/next-pwa`, Next.js App Router, icônes SVG/PNG générées depuis le wordmark Axiom. Dépend du déploiement Vercel HTTPS actif (Phase 2) — les PWA requièrent HTTPS.

## Hors-périmètre

- Mode offline complet (cache des données utilisateur, sync en arrière-plan) — le caching des assets statiques suffit.
- Push notifications natives — hors scope.
